# simple-pdf-editor
rotate, split, and merge

## vibe coding 流程
1. 明確需求與邊界  
   - 只做「本機」PDF 操作：旋轉、切割、合併。  
   - 完全前端實作，不上傳檔案、不依賴後端。

2. 選擇技術堆疊  
   - 使用 pdf.js 負責 PDF 預覽與翻頁。  
   - 使用 pdf-lib 負責 PDF 的結構修改與輸出。

3. 建立最小可行版本（MVP）  
   - 完成「載入單一 PDF → 預覽 → 旋轉指定頁 → 下載新 PDF」流程。  
   - 驗證檔案讀取、預覽渲染、旋轉與下載等核心技術是否可行。

4. 功能擴充：切割與合併  
   - 切割：利用 `copyPages` 取出指定頁面，組成新 PDF 並下載。  
   - 合併：將多個 PDF 逐一載入，使用 `copyPages` 複製到同一個目標文件中。

5. UX / UI 重構  
   - 將操作整理為單一流程：  
     1) 選工具 → 2) 選檔 → 3) 設定 → 4) 執行。  
   - 以 Segmented Control 切換「旋轉 / 取出 / 合併」，共用同一組預覽區與主操作按鈕。

6. 行為精緻化  
   - 合併模式：  
     - 選擇多檔後立即在記憶體中合併，預覽的即為合併後結果。  
     - 用陣列維護檔案清單，呈現檔名列表，並提供 `✕` 按鈕移除單一檔案後重新合併。  
   - 檔名策略：  
     - 下載檔名統一採用 `[原檔名]_[動作].pdf`，例如 `mydoc_rotated.pdf`、`mydoc_extracted.pdf`、`first_merged.pdf`。

7. 部署與分享考量  
   - 將整個工具設計為純靜態網站（HTML + JS），可直接部署到 GitHub Pages 或任意靜態空間。  
   - 在說明中強調所有操作都在瀏覽器本機完成，保障使用者檔案隱私。
